OWF R1 Documentation

 

 

OWF (Offgrid Weather Forecaster) is an offline, sensor-based weather prediction system powered by the Meshtastic® ecosystem. This page covers everything needed to build, use, and understand the OWF R1 device.

 

 

1. Overview

The OWF device is designed for low-power, autonomous forecasting without internet access. Based on the Zambretti forecasting algorithm and enhanced with custom prediction logic, it offers accurate, decentralized weather reporting.

2. Features

  • 100% off-grid operation
  • Forecasts based on pressure, trend, and other sensor data
  • Meshtastic® integration for distributed sensor networks
  • Daily automatic forecasts and on-demand reports
  • Expandable via Arduino-compatible headers

3. Hardware Specifications

MCU:Arduino Pro Mini 3.3V / 8 MHz
Sensor:BMP280 or BME280
Power:LiPo or solar panel (5V input)
Radio:LoRa via Meshtastic® (ESP32-based board)
Enclosure:3D-printed, weather-resistant shell

4. Assembly Guide

  1. Solder the BMP/BME280 sensor to the I2C pins on the Arduino
  2. Connect power via a 3.3V regulator or directly to VCC
  3. Flash firmware using FTDI adapter (set to 3.3V)
  4. Mount in weather-resistant enclosure with vents for pressure readings

5. Firmware & Flashing

Flash using the Arduino IDE or PlatformIO. Ensure board is set to "Arduino Pro Mini (3.3V, 8 MHz)". The firmware is available on GitHub:

→ OWF Firmware Repository

6. Usage Instructions

  • Start the OWF and allow the barometer to stabilize
  • Every morning at 5:00 AM (or manually), it will compare trends
  • Forecast will be sent via Meshtastic® with symbols (sun, rain, storm)

7. Networking via Meshtastic®

OWF units communicate using the Meshtastic® mesh protocol. This allows:

  • Distributed forecasts
  • Fallback communication in emergencies
  • Multi-point environmental coverage

8. Forecast Algorithm

Forecasts are generated using:

  • Zambretti Forecaster: Pressure-based rule system
  • Trend Analysis: Checks short-term pressure change rate
  • Humidity/Temperature Adjustments (if BME280 is used)

9. Troubleshooting

  • No forecast sent? Check power and Meshtastic config
  • Wrong predictions? Let the barometer run for 24h before trusting it
  • Flashing fails? Confirm you selected 3.3V in the FTDI adapter
