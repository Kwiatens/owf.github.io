1. Introduction

The OWF (Offgrid Weather Forecaster) is a standalone weather forecasting device that uses sensor data and local algorithms to predict weather conditions without any internet access. It is powered by meshtastic® and designed for low-power, remote operation.

2. Features

  • Completely offgrid – no Wi-Fi, no GSM, no internet required
  • Uses pressure trends and algorithms like Zambretti for prediction
  • Supports Meshtastic® radio communication
  • Forecast sharing between devices
  • Low power consumption, solar-compatible

3. Hardware Requirements

Component Example
Microcontroller Arduino Pro Mini 3.3V
Sensor BMP280 or BME280
Radio Module T-Echo, T-BEAM, or similar LoRa boards

4. Example Code


// Calculate pressure trend
float pressureDelta = currentPressure - previousPressure;
if (pressureDelta > 0.5) {
  forecast = "Improving weather";
} else if (pressureDelta < -0.5) {
  forecast = "Worsening weather";
} else {
  forecast = "Stable weather";
}

5. Usage Tips

Mount the OWF unit away from direct sunlight for more accurate temperature and pressure readings. Ensure the sensor is in a weatherproof, ventilated enclosure. For best forecasting results, keep the device powered on continuously so it can gather and compare long-term pressure data.

6. Troubleshooting

  • No forecast? – Ensure the sensor is properly connected and not reporting null values.
  • Time issues? – OWF uses internal timers only. Be sure the board has been on long enough for comparisons.
  • LoRa not working? – Check antenna and radio board configuration.
